# 校园二手交易平台 — 分阶段开发计划

> 总工期预估：8~10 周（按每天 2-4 小时投入）
> 每个阶段产出可独立运行、可演示的版本

---

## 阶段总览

| 阶段 | 名称 | 周期 | 产出物 |
|------|------|------|--------|
| P0 | 项目初始化 | 1 周 | 空白小程序 + 云开发环境 |
| P1 | 用户体系 + 商品发布 | 2 周 | 可登录、可发商品 |
| P2 | 商品浏览 + 搜索 | 2 周 | 可逛、可搜、可筛选 |
| P3 | 收藏 + 私信聊天 | 2 周 | 可互动、可沟通 |
| P4 | 交易闭环 + 评价 | 1-2 周 | 可完成交易流程 |
| P5 | 体验优化 + 论文素材 | 1 周 | 可演示的完整产品 |

---

## P0 — 项目初始化（第 1 周）

### 目标
搭建开发环境，跑通最小可用链路。

### 任务清单
- [ ] 注册微信小程序 AppID（如果已有则跳过）
- [ ] 微信开发者工具创建项目
- [ ] 开通微信云开发环境
- [ ] 创建云数据库集合（基础 4 个空集合：users / goods / categories / messages）
- [ ] 配置底部 TabBar（首页 / 分类 / 发布 / 消息 / 我的）
- [ ] 创建 5 个 Tab 页面的基础骨架（空页面）
- [ ] 配置 `project.config.json`（appid、云开发根目录等）
- [ ] 提交 git 仓库

### 验证标准
- 项目在开发者工具中正常运行
- 底部 Tab 可切换
- 云开发控制台可访问

---

## P1 — 用户体系 + 商品发布（第 2-3 周）

### 目标
完成用户登录流程和商品发布功能。

### 任务清单

#### 用户模块
- [ ] 实现 `wx.login()` → 云函数获取 openid
- [ ] 用户表写入（首次登录自动注册）
- [ ] 个人中心页面（展示头像、昵称、学校）
- [ ] 用户信息编辑页面（修改昵称、学校、联系方式）
- [ ] 手机号快捷绑定（可选，用 `getPhoneNumber`）

#### 商品发布
- [ ] 发布页面 UI（标题、描述、价格、原价、分类选择器、成色选择、校区选择）
- [ ] 多图片上传（调用云存储，最多 9 张）
- [ ] 提交表单 → 写入 goods 集合
- [ ] 表单校验（必填项、价格格式、图片数量）
- [ ] 发布成功后跳转到商品详情页

### 数据库操作
- [ ] 创建云函数：`login`（获取 openid + 注册用户）
- [ ] 创建云函数：`uploadImage`（上传到云存储）
- [ ] 创建云函数：`createGoods`（发布商品）

### 验证标准
- 用户可微信一键登录，在个人中心看到自己的信息
- 用户可完整走完发布商品的流程
- 云数据库中能看到新写入的用户和商品记录

---

## P2 — 商品列表 + 搜索（第 4-5 周）

### 目标
用户能浏览商品、搜索商品、查看详情。

### 任务清单

#### 首页商品流
- [ ] 首页瀑布流/列表展示（按发布时间倒序）
- [ ] 下拉加载更多（分页，每页 10 条）
- [ ] 顶部轮播 Banner（运营位/公告）
- [ ] 分类快捷入口（横向滚动 icon 导航）

#### 商品详情页
- [ ] 图片轮播（swiper）
- [ ] 商品信息展示（标题、价格、原价、成色、描述）
- [ ] 卖家信息卡片（头像、昵称、学校、信用分）
- [ ] "想要"按钮 + "收藏"按钮
- [ ] 浏览计数 +1

#### 分类页
- [ ] 分类网格展示（从 categories 集合读取）
- [ ] 点击分类 → 进入该分类商品列表

#### 搜索页
- [ ] 搜索框（历史记录、热门搜索）
- [ ] 搜索结果列表
- [ ] 筛选面板（分类 + 价格区间 + 校区 + 成色）
- [ ] 排序切换（最新 / 价格低到高 / 价格高到低）

### 数据库操作
- [ ] 创建云函数：`getGoodsList`（分页 + 排序 + 筛选）
- [ ] 创建云函数：`getGoodsDetail`（商品详情 + 卖家信息）
- [ ] 创建云函数：`getCategories`（获取分类列表）
- [ ] 创建云函数：`searchGoods`（关键词搜索 + 筛选）

### 验证标准
- 首页能展示商品列表，下拉加载更多正常
- 点击商品进入详情页，信息完整
- 搜索能根据关键词返回结果
- 分类筛选和排序正常

## P3 — 收藏 + 私信聊天（第 6-7 周）

### 目标
用户可收藏商品，并与卖家在线沟通。

### 任务清单

#### 收藏功能
- [ ] 商品详情页收藏/取消收藏（心形按钮动画）
- [ ] 收藏列表页面（展示所有收藏的商品）
- [ ] 收藏状态在商品列表中可见

#### 私信聊天
- [ ] 消息列表页面（展示所有会话，按最后消息时间排序）
- [ ] 聊天详情页面（展示历史消息 + 输入框）
- [ ] 文字消息发送
- [ ] 图片消息发送
- [ ] 商品卡片消息（从详情页发起时携带商品信息）
- [ ] 未读消息红点提示
- [ ] 会话列表展示最后一条消息摘要

> 聊天方案选一：
> - 方案 A（推荐毕设）：云函数定时轮询 messages 集合 + 触发式读取
> - 方案 B：使用 WebSocket 云函数实现实时推送
> - 方案 C：接入第三方 IM SDK（环信/融云等）

### 数据库操作
- [ ] 创建云函数：`toggleFavorite`（收藏/取消）
- [ ] 创建云函数：`getFavoriteList`（获取收藏列表）
- [ ] 创建云函数：`sendMessage`（发送消息）
- [ ] 创建云函数：`getConversationList`（获取会话列表）
- [ ] 创建云函数：`getMessages`（获取聊天记录）

### 验证标准
- 商品详情页可收藏/取消收藏，收藏列表展示正确
- 从商品详情页可发起私信，携带商品信息
- 双方可互相发送文字/图片消息
- 消息列表有红点提示，显示未读

## P4 — 交易闭环 + 评价（第 8-9 周）

### 目标
完成交易流程闭环：想要→联系→成交标记→评价。

### 任务清单

#### 交易流程
- [ ] "想要"按钮 → 自动创建会话并推送商品卡片
- [ ] 卖家确认交易 → 标记商品为"已售出"
- [ ] 交易记录页面（买家视角：我买到的；卖家视角：我卖出的）
- [ ] 商品详情页显示"已售出"状态

#### 评价体系
- [ ] 交易完成后互评入口（14 天内有效）
- [ ] 评价页面（评分 1-5 星 + 文字评价）
- [ ] 卖家信用分计算（根据评价加权）
- [ ] 评价在卖家主页展示

#### 辅助功能
- [ ] 商品下架（卖家在"我的发布"中操作）
- [ ] 重新上架（下架的商品可再次发布）
- [ ] 举报商品（违规举报 → 写入举报记录）

### 数据库操作
- [ ] 创建云函数：`markAsSold`（标记已售）
- [ ] 创建云函数：`submitReview`（提交评价）
- [ ] 创建云函数：`getUserReviews`（获取评价列表）
- [ ] 创建云函数：`reportGoods`（举报商品）

### 验证标准
- 点击"想要"后卖家收到消息通知
- 卖家可标记商品已售出，商品详情页正确显示状态
- 交易双方可互评，评分影响信用分
- 卖家可下架/重新上架商品

## 开发建议

### 关于聊天方案的选择

| 方案 | 优点 | 缺点 | 推荐场景 |
|------|------|------|----------|
| A. 轮询拉取 | 实现简单，云函数免费额度够用 | 实时性略差（5-10s 延迟） | ✅ **毕设推荐** |
| B. WebSocket | 实时推送 | 需维护长连接，复杂度高 | 追求体验 |
| C. 第三方 IM | 功能完善 | 有费用、接入复杂度 | 商业项目 |

### 关于后端方案的选择

| 方案 | 优点 | 缺点 | 推荐场景 |
|------|------|------|----------|
| 微信云开发 | 无需自建服务器、快速上线、免费额度足够毕设 | 灵活性受限 | ✅ **毕设首选** |
| Node.js + MySQL | 完全可控、面试含金量高 | 需自行部署服务器 | 想练后端 |
| 小程序 + 自建 API | 前后端分离、扩展性强 | 开发周期长 | 团队项目 |
