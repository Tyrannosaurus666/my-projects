# 编程代码注释智能生成与优化助手 — 项目开发方案（精简版）

---

## 一、项目总览

| 维度 | 说明 |
|---|---|
| **核心目标** | 用户上传代码文件 → 系统通过 **AST语法分析** + **大模型** 两种方式自动生成注释 → 网页预览 + 下载 |
| **输入** | Python / C++ 代码文件（单文件或批量） |
| **输出** | 带高质量注释的代码文件，可在线预览、可下载 |
| **核心价值** | AST 提供结构级注释（函数/类/分支），LLM 提供语义级注释（意图/逻辑/易错点），两者互补 |
| **无登录** | 不需要用户注册/登录，打开即用 |

```
┌─────────────────────────────────────────────────────────┐

└──────┬──────────────────────────────────────┬───────────┘
       │                                      │
┌──────▼──────────────┐              ┌───────▼───────────┐
│   AST 语法分析引擎    │              │   大模型注释引擎     │
│  (Python ast 标准库) │              │  (DeepSeek API)    │
│                     │              │                    │
│ · 函数/类结构提取    │              │ · 函数级注释         │
│ · 循环/条件分支标注  │              │ · 核心逻辑解释       │
│ · 变量/参数识别      │              │ · 易错点标注         │
```

**简化说明：**
- Python 用 `ast` 标准库做语法树分析（零额外依赖）
- C++ 用正则做基本函数/类识别 + DeepSeek API 做语义注释（不需要 tree-sitter / Clang）
- 不引入数据库，用文件系统 + 内存字典存储任务状态
- 前端用原生三件套，不引入框架（适合期末作业快速交付）

## 四、后端 API 设计（无认证）

### 4.1 API 列表

| 方法 | 路径 | 功能 | 请求 | 响应 |
|---|---|---|---|---|
| `POST` | `/upload` | 上传代码文件 | `multipart/form-data` (file) | `{ file_id, filename, language }` |
| `POST` | `/analyze/{file_id}` | 触发分析任务 | 无 | `{ task_id, status: "processing" }` |
| `GET` | `/analyze/{file_id}/status` | 查询分析进度 | 无 | `{ status, ast_done, llm_done }` |
| `GET` | `/result/{file_id}` | 获取分析结果 | 无 | `{ ast_result, llm_result }` |
| `GET` | `/download/{file_id}` | 下载带注释代码 | `?type=ast 或 llm` | 文件流（.py / .cpp） |

### 4.2 数据处理流程

```
POST /upload
    ↓
保存文件到 uploads/ → 检测语言 → 返回 file_id
    ↓
POST /analyze/{file_id}
    ↓
    ├─→ AST 分析线程：读取代码 → ast.parse → 遍历节点生成注释 → 存储结果
    └─→ LLM 分析线程：读取代码 → 拼装 Prompt → 调用 DeepSeek API → 存储结果
    ↓
GET /result/{file_id} → 返回 ast_result + llm_result
```

## 六、AST 分析引擎设计

### 6.1 Python AST 分析（ast_analyzer.py）

使用 Python 标准库 `ast` 模块，无需额外安装。

**分析维度：**

| 分析项 | AST 节点 | 生成注释 |
|---|---|---|
| 函数定义 | `ast.FunctionDef` | 函数功能描述 + 参数说明 + 返回值说明 |
| 类定义 | `ast.ClassDef` | 类的职责说明 + 方法列表 |
| `if/elif/else` 分支 | `ast.If` | 该分支的判断逻辑说明 |
| `for/while` 循环 | `ast.For` / `ast.While` | 循环目的说明 |
| 递归调用检测 | 函数内调用自身 | 标注 `# 注意：此函数使用了递归` |
| 已有注释检测 | 任何 ast 节点的行号 | 标记"已有注释"，避免重复添加 |
| 变量赋值 | `ast.Assign` | 仅对复杂赋值（如列表推导）加注释 |

**核心逻辑伪代码：**

```python
import ast

def analyze_python_code(source_code: str) -> dict:
    tree = ast.parse(source_code)
    result = {
        "functions": [],
        "classes": [],
        "branches": [],
        "loops": [],
        "has_recursion": False
    }

    class Analyzer(ast.NodeVisitor):
        def visit_FunctionDef(self, node):
            # 提取函数名、参数、是否递归
            func_info = {
                "name": node.name,
                "lineno": node.lineno,
                "args": [a.arg for a in node.args.args],
                "body_summary": get_body_summary(node),
                "comment": generate_func_comment(node)  # 自动生成的注释文本
            }
            result["functions"].append(func_info)
            self.generic_visit(node)

        def visit_ClassDef(self, node):
            result["classes"].append({
                "name": node.name,
                "lineno": node.lineno,
                "methods": [n.name for n in node.body if isinstance(n, ast.FunctionDef)]
            })
            self.generic_visit(node)

        # ... visit_If, visit_For, visit_While 类似

    Analyzer().visit(tree)
    return result
```

### 6.2 C++ 简易分析（cpp_parser.py）

C++ 语法复杂，不用完整解析器。采用**正则 + LLM 兜底**策略：

1. 用正则提取函数签名（`\w+\s+\w+\([^)]*\)\s*\{`）
2. 用正则提取类声明（`class\s+\w+`）
3. 结构信息作为 Prompt 的上下文传给 LLM
4. 注释生成完全依赖 LLM

```python
import re

def analyze_cpp_code(source_code: str) -> dict:
    functions = re.findall(
        r'(\w+(?:\s+\w+)*)\s+(\w+)\(([^)]*)\)\s*\{',
        source_code
    )
    classes = re.findall(r'class\s+(\w+)', source_code)

    return {
        "functions": [{"return_type": r, "name": n, "params": p} for r, n, p in functions],
        "classes": [{"name": c} for c in classes],
        "note": "C++ 语义注释由 LLM 生成"
    }
```

## 七、大模型注释生成引擎设计

### 7.1 为什么选 DeepSeek API

- 性价比高（适合学生作业成本控制）
- 代码理解能力强
- 兼容 OpenAI SDK，接入简单
- 有免费额度

### 7.2 Prompt 设计（prompt.py）

```python
SYSTEM_PROMPT = """你是一个编程教学助手。你的任务是为学生代码添加清晰、有用的注释。

## 注释规范（必须严格遵守）：
1. 每个函数/类定义上方必须添加功能说明注释
2. 复杂逻辑块（>5行）上方添加块注释说明意图
3. 禁止对明显自明的语句加注释（如 x = 1、print(x)）
4. 对易错/关键语句用 "# 注意：" 标注
5. 已有注释的代码行不要重复添加注释
6. 直接返回带注释的完整代码，不要任何额外解释

## 注释风格：
- 函数注释："""简短描述功能，参数和返回值"""
- 行内注释：# 简洁说明意图
- 块注释：# === 某功能块 ===
"""

FEW_SHOT_EXAMPLE = """
# 原始代码：
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)

# 注释后代码：
def factorial(n):
    """计算 n 的阶乘（n!），使用递归实现。"""
    # 注意：此函数使用了递归，n 过大可能导致栈溢出
    if n <= 1:
        return 1
    return n * factorial(n - 1)
"""
```

### 7.3 API 调用封装（client.py）

```python
from openai import OpenAI

client = OpenAI(
    api_key="你的DeepSeek_API_Key",
    base_url="https://api.deepseek.com"
)

def generate_comments(code: str, language: str, ast_hints: dict = None) -> str:
    """调用 DeepSeek API 生成带注释的代码"""
    user_prompt = f"请为以下 {language} 代码添加注释：\n\n```{language}\n{code}\n```"

    if ast_hints:
        hint_text = f"\n\nAST 分析提示（辅助你更精准地注释）：\n{ast_hints}"
        user_prompt += hint_text

    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "user", "content": user_prompt},
        ],
        temperature=0.2,
        max_tokens=4096,
    )

    return response.choices[0].message.content
```

## 十二、关键技术点备忘

| 关键点 | 实现方式 |
|---|---|
| **Python AST 不解析注释** | `ast.parse` 默认跳过注释，如需检测已有注释需手动解析源码行 |
| **LLM 返回码块提取** | LLM 返回可能含 markdown 代码块标记，需正则提取 \`\`\` 之间的内容 |
| **CORS 跨域** | FastAPI 添加 CORS 中间件允许前端跨域访问 |
| **并发分析** | 使用 `asyncio` 或线程池，AST 和 LLM 分析可并行执行 |
| **成本控制** | 每段代码预估 token 数约 500~2000，100 段全量约 10 万 token，按 DeepSeek 价格约 ¥0.1~0.2 |
| **错误处理** | LLM 调用失败重试 1 次，超时 30 秒 |
