## 一、项目概述

### 1.1 项目简介
**招聘管理与人才看板系统**是一个现代化的企业招聘管理平台，旨在优化招聘流程、提升招聘效率。系统支持候选人、HR和管理员三种角色，覆盖从职位发布到录用决策的全流程管理。

### 1.2 核心功能
- **候选人端**：职位浏览搜索、简历投递、申请状态跟踪
- **HR端**：职位发布管理、简历筛选、面试安排、候选人状态更新
- **管理端**：系统配置、权限管理、数据统计
- **看板功能**：招聘漏斗可视化、数据统计图表

### 1.3 项目特点
- **全栈实现**：前后端分离，功能完整
- **多角色权限**：精细化的权限控制体系
- **状态机管理**：规范的招聘流程状态流转
- **数据可视化**：招聘数据统计与图表展示
- **文件管理**：简历上传与存储

## 四、数据库设计

### 4.1 核心数据表
| 表名 | 说明 | 优先级 |
|------|------|--------|
| user | 用户表（候选人、HR、管理员） | P0 |
| job_post | 职位发布表 | P0 |
| application | 职位申请表 | P0 |
| interview_note | 面试备注表 | P0 |
| job_tag | 职位标签表 | P1 |
| application_stage | 申请阶段历史表 | P1 |
| interview_schedule | 面试日程表 | P2 |

### 4.2 数据关系
- 一个用户可以发布多个职位（HR）
- 一个职位可以收到多个申请
- 一个申请可以有多个面试备注
- 一个申请有状态流转历史记录

## 五、API接口设计

### 5.1 接口概览
系统共提供21个RESTful API，分为三个优先级：

- **P0 API（12个）**：核心功能，包括用户认证、职位管理、申请管理
- **P1 API（3个）**：扩展功能，包括数据统计、标签筛选
- **P2 API（6个）**：高级功能，包括面试日程、Offer生成

### 5.2 主要接口分类
1. **认证接口**：`/api/auth/*`
2. **职位接口**：`/api/jobs/*`
3. **申请接口**：`/api/applications/*`
4. **统计接口**：`/api/statistics/*`
5. **日程接口**：`/api/schedules/*`

## 七、部署与运行

### 7.1 环境要求
- **Java**：JDK 21+
- **Node.js**：24.0.0+
- **MySQL**：8.0+
- **Redis**：3.2.100+（可选）
- **Maven**：3.6.0+
- **pnpm**：10.33.4+

### 7.2 启动步骤
1. **启动Redis**（可选）
2. **初始化数据库**：执行`sql/01-init.sql`
3. **启动后端**：`cd backend && mvn spring-boot:run`
4. **启动前端**：`cd frontend && pnpm install && pnpm dev`
5. **访问系统**：前端 `http://localhost:5173`，后端 `http://localhost:8080`
